# Обработчики событий

Обработчики событий позволяют выполнять произвольный код в определённых точках жизненного цикла бенчмарков - например, для подготовки данных, управления ресурсами или анализа результатов, не засоряя при этом измеряемый код.

## Жизненный цикл

```
ПередВсеми
│
├─ [Кейс 1]
│   ├─ ПередКаждым
│   ├─ Итерации бенчмарка
│   └─ ПослеКаждого
│
├─ [Кейс 2]
│   ├─ ПередКаждым
│   ├─ Итерации бенчмарка
│   └─ ПослеКаждого
│
└─ ПослеВсех
```

> **Кейс** - одна комбинация бенчмарка и набора параметров. Если у вас 2 бенчмарка и 3 набора параметров, то будет 6 кейсов, для каждого из которых вызываются `ПередКаждым` и `ПослеКаждого`.

## Аннотации

### `&ПередВсеми`

Вызывается **один раз перед запуском всех кейсов**. Используется для инициализации разделяемых ресурсов: загрузки данных, открытия соединений, настройки окружения.

```bsl
// Параметры:
//   * Контекст - Структура:
//     ** Конфигурация          - КонфигурацияБенчмарков          - Текущая конфигурация запуска
//     ** ДескрипторыБенчмарков - КоллекцияДескрипторовБенчмарков - Список запланированных бенчмарков
&ПередВсеми
Процедура Инициализация(Контекст) Экспорт
```

### `&ПослеВсех`

Вызывается **один раз после завершения всех кейсов**. Используется для освобождения ресурсов, записи итоговой статистики, сохранения или отправки результатов.

```bsl
// Параметры:
//   * Контекст - Структура:
//     ** Конфигурация          - КонфигурацияБенчмарков           - Текущая конфигурация запуска
//     ** ДескрипторыБенчмарков - КоллекцияДескрипторовБенчмарков  - Список бенчмарков
//     ** Запуски               - Массив из РезультатЗапускаКейса  - Результаты всех запущенных кейсов
//     ** Отчет                 - ОтчетБенчмарков                  - Сформированный отчет
//     ** СредаОкружения        - СредаОкруженияБенчмарков         - Информация о среде выполнения
&ПослеВсех
Процедура Завершение(Контекст) Экспорт
```

### `&ПередКаждым`

Вызывается **перед каждым кейсом**. Используется для сброса состояния, генерации тестовых данных с учётом параметров текущего кейса.

```bsl
// Параметры:
//   * Контекст - Структура:
//     ** Кейс - КейсБенчмарка - Описание текущего запускаемого кейса
&ПередКаждым
Процедура ПодготовитьКейс(Контекст) Экспорт
```

### `&ПослеКаждого`

Вызывается **после каждого кейса**. Используется для постобработки: валидации результатов, освобождения ресурсов конкретного кейса, логирования.

```bsl
// Параметры:
//   * Контекст - Структура:
//     ** Кейс       - КейсБенчмарка                    - Описание завершённого кейса
//     ** Замеры     - Массив из РезультатИтерацииКейса - Результаты отдельных итераций
//     ** Статистика - СтатистикаБенчмарка              - Итоговая статистика кейса
&ПослеКаждого
Процедура ОбработатьРезультатКейса(Контекст) Экспорт
```

## API

Обработчики можно регистрировать программно через объект `КонфигурацияБенчмарков`.

```bsl
// Регистрация обработчика из текущего класса
Конфигурация.ДобавитьОбработчикСобытия("МойОбработчик", СобытияБенчмарков.ПослеВсех);

// Регистрация обработчика из другого модуля или класса
Конфигурация.ДобавитьОбработчикСобытия("МодульОбработчиков.МойОбработчик", СобытияБенчмарков.ПослеВсех);

// Получение всех зарегистрированных обработчиков
Обработчики = Конфигурация.ОбработчикиСобытий();

// Получение обработчиков конкретного события
ОбработчикиПослеВсех = Конфигурация.ОбработчикиСобытия(СобытияБенчмарков.ПослеВсех);

// Удаление обработчиков события
Конфигурация.УдалитьОбработчикиСобытия(СобытияБенчмарков.ПередКаждым);

// Удаление всех обработчиков
Конфигурация.УдалитьОбработчикиСобытий();
```

## Примеры

### Пример 1: Разделяемые данные между бенчмарками

Данные инициализируются один раз в `ПередВсеми` и переиспользуются всеми бенчмарками.

```bsl
Перем ДвоичныеДанные;

&ПередВсеми
Процедура Инициализация(Контекст) Экспорт
	ДвоичныеДанные = ПолучитьДвоичныеДанныеИзСтроки("Тестовые данные для хеширования");
КонецПроцедуры

&ПослеВсех
Процедура Завершение(Контекст) Экспорт
	ДвоичныеДанные = Неопределено;
КонецПроцедуры

&Бенчмарк
Процедура SHA256() Экспорт
	ХешированиеДанных = Новый ХешированиеДанных(ХешФункция.SHA256);
	ХешированиеДанных.Добавить(ДвоичныеДанные);
	ХешСумма = ХешированиеДанных.ХешСумма;
КонецПроцедуры

&Бенчмарк
Процедура MD5() Экспорт
	ХешированиеДанных = Новый ХешированиеДанных(ХешФункция.MD5);
	ХешированиеДанных.Добавить(ДвоичныеДанные);
	ХешСумма = ХешированиеДанных.ХешСумма;
КонецПроцедуры
```

### Пример 2: Подготовка данных с учётом параметров кейса

`ПередКаждым` позволяет подготовить данные под конкретный кейс - например, массив нужного размера.

```bsl
&Параметры(100, 1000, 10000)
Перем Количество Экспорт;

Перем Данные;

&ПередКаждым
Процедура ПодготовитьДанные(Контекст) Экспорт
	Данные = Новый Массив();
	Для Число = 1 По Количество Цикл
		Данные.Добавить(Число);
	КонецЦикла;
КонецПроцедуры

&ПослеКаждого
Процедура ПроверитьРезультат(Контекст) Экспорт
	Контекст.Статистика.ВМиллисекунды();
	Если Контекст.Статистика.Среднее > 1 Тогда // мс
		Сообщить(СтрШаблон("Внимание: кейс с Количество=%1 выполняется медленно", Количество));
	КонецЕсли;
КонецПроцедуры

&Бенчмарк
Процедура СуммаЭлементов() Экспорт
	Сумма = 0;
	Для Каждого Число Из Данные Цикл
		Сумма = Сумма + Число;
	КонецЦикла;
КонецПроцедуры
```

### Пример 3: Сохранение результатов после прогона

Используйте `ПослеВсех` для доступа к итоговому отчёту и экспорта результатов.

```bsl
&ПослеВсех
Процедура СохранитьРезультаты(Контекст) Экспорт

	Для Каждого Запуск Из Контекст.Запуски Цикл
		Дескриптор = Запуск.Кейс.ДескрипторБенчмарка();
		Статистика = Запуск.Статистика;
		Сообщить(СтрШаблон("%1: среднее = %2 нс, op/s = %3",
			Дескриптор.Метод(),
			Формат(Статистика.Среднее, "ЧДЦ=2"),
			Формат(Статистика.ОперацийВСекунду, "ЧГ=")
		));
	КонецЦикла;

КонецПроцедуры

&Бенчмарк
Процедура МойБенчмарк() Экспорт
	// Логика бенчмарка
КонецПроцедуры
```
