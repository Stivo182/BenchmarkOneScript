# Аннотации

Документ описывает аннотации, которые BenchmarkOneScript понимает в классах бенчмарков.

Аннотации позволяют декларативно:

- помечать методы как бенчмарки;
- задавать параметры и источники данных;
- настраивать стратегию выполнения и отчет;
- подключать дополнительные runtime-среды;
- встраивать обработчики событий жизненного цикла.


## Основные аннотации бенчмарков

### `&Бенчмарк`

**Синтаксис**

`&Бенчмарк(<Наименование> = "")`

**Где указывается**

На экспортном методе бенчмарка.

Помечает метод как измеряемый бенчмарк.

**Параметры**

- `Наименование` - пользовательское имя бенчмарка для отчета и консольного вывода.

```bsl
&Бенчмарк
Процедура СортировкаВставками() Экспорт
КонецПроцедуры
```

```bsl
&Бенчмарк(Наименование = "Сортировка вставками")
Процедура СортировкаВставками() Экспорт
КонецПроцедуры
```

Если `Наименование` задано, оно используется в колонке `Method` отчета вместо имени метода.

### `&Эталон`

**Синтаксис**

`&Эталон`

**Где указывается**

На экспортном методе бенчмарка.

Помечает метод как эталонный бенчмарк для расчета сравнительных метрик вроде `Ratio`.

```bsl
&Бенчмарк
&Эталон
Процедура БазовыйАлгоритм() Экспорт
КонецПроцедуры
```

Подробнее: [Эталоны](Эталоны.md).

### `&Категория`

**Синтаксис**

`&Категория(<Категория>)`

**Где указывается**

На экспортном методе бенчмарка.

Назначает категорию бенчмарку. Категории используются для группировки и выбора эталонов.

**Параметры**

- `Категория` - строковое имя группы, к которой относится бенчмарк.

```bsl
&Бенчмарк
&Категория("Сортировки")
Процедура БыстраяСортировка() Экспорт
КонецПроцедуры
```

## Параметризация

### `&Параметры`

**Синтаксис**

`&Параметры(<Значение1>, <Значение2>, ...)`

**Где указывается**

- На экспортном поле для задания значений одного параметра.
- На экспортном методе бенчмарка для задания наборов аргументов.

Задает значения параметров бенчмарка.

```bsl
&Параметры(100, 1000, 10000)
Перем РазмерМассива Экспорт;
```

```bsl
&Бенчмарк
&Параметры(100, Истина)
&Параметры(1000, Ложь)
Процедура Тест(Размер, СлучайныеДанные) Экспорт
КонецПроцедуры
```

Подробнее: [Параметризация](Параметризация.md).

### `&ИсточникПараметров`

**Синтаксис**

`&ИсточникПараметров(<Источник>)`

**Где указывается**

- На экспортном поле для задания значений одного параметра.
- На экспортном методе бенчмарка для задания наборов аргументов.

Подключает экспортную функцию, которая возвращает значения параметров.

**Параметры**

- `Источник` - строковая ссылка на экспортный метод-источник данных.

Поддерживаются ссылки вида:

- `"ИмяМетода"` - метод текущего класса;
- `"ИмяТипа.ИмяМетода"` - внешний модуль или класс.

Если бенчмарк принимает несколько параметров, источник может возвращать:

- `Массив` значений по порядку;
- `Структура` или `Соответствие` с ключами по именам параметров;
- `СтрокаТаблицыЗначений`.

```bsl
&ИсточникПараметров("ПолучитьРазмеры")
Перем Размер Экспорт;

Функция ПолучитьРазмеры() Экспорт
	Размеры = Новый Массив();
	Размеры.Добавить(100);
	Размеры.Добавить(1000);
	Размеры.Добавить(10000);
	Возврат Размеры;
КонецФункции
```

```bsl
&Бенчмарк
&ИсточникПараметров("ПолучитьНаборыПараметров")
Процедура Тест(Размер, Название) Экспорт
КонецПроцедуры
```

Подробнее: [Параметризация](Параметризация.md).

## Аннотации конфигурации запуска

Эти аннотации указываются на `ПриСозданииОбъекта()`.

### Стратегия и итерации

| Аннотация | Назначение |
| --- | --- |
| `&СтратегияПропускнаяСпособность` | Throughput-режим: прогрев и измерения. |
| `&СтратегияХолодныйЗапуск` | Cold start-режим: измерение первого запуска. |
| `&КоличествоИтераций(<КоличествоИтераций>)` | Количество измерительных итераций. |
| `&КоличествоПрогревочныхИтераций(<КоличествоПрогревочныхИтераций>)` | Количество прогревочных итераций. |
| `&КоличествоВызововЗаИтерацию(<КоличествоВызововЗаИтерацию>)` | Число вызовов метода за итерацию. |
| `&МинимальноеВремяИтерации(<МинимальноеВремяИтерации>)` | Минимальная длительность итерации в миллисекундах. |
| `&МинимальноеКоличествоВызововЗаИтерацию(<КоличествоВызововЗаИтерацию>)` | Нижняя граница числа вызовов за итерацию. |

```bsl
&СтратегияПропускнаяСпособность
&КоличествоИтераций(15)
&КоличествоПрогревочныхИтераций(6)
&МинимальноеВремяИтерации(200)
Процедура ПриСозданииОбъекта()
КонецПроцедуры
```

Примечания:

- `&КоличествоПрогревочныхИтераций`, `&КоличествоВызововЗаИтерацию`,
  `&МинимальноеВремяИтерации` и
  `&МинимальноеКоличествоВызововЗаИтерацию` игнорируются в `ХолодныйЗапуск`.
- Если `&КоличествоВызововЗаИтерацию(0)`, число вызовов рассчитывается автоматически.

Подробнее: [Стратегии запуска](СтратегииЗапуска.md), [Управление итерациями и вызовами](УправлениеИтерациямиИВызовами.md).

### Отчет и экспорт

| Аннотация | Назначение |
| --- | --- |
| `&МониторингПамяти` | Включает сбор выделений памяти. |
| `&СортировкаПоМетоду` | Сортирует отчет по имени метода. |
| `&СортировкаОтБыстрыхКМедленным` | Сортирует отчет по времени от быстрых к медленным. |
| `&СортировкаОтМедленныхКБыстрым` | Сортирует отчет по времени от медленных к быстрым. |
| `&КолонкаМин` | Добавляет колонку `Min`. |
| `&КолонкаМакс` | Добавляет колонку `Max`. |
| `&КолонкаМедиана` | Добавляет колонку `Median`. |
| `&КолонкаНижнийКвартиль` | Добавляет колонку нижнего квартиля. |
| `&КолонкаВерхнийКвартиль` | Добавляет колонку верхнего квартиля. |
| `&КолонкаПроцентиль(<Процентиль1>, <Процентиль2>, ...)` | Добавляет одну или несколько процентильных колонок. |
| `&КолонкаRatioSD` | Добавляет колонку `RatioSD`. |
| `&ЭкспортMarkdown` | Включает экспорт в Markdown. |
| `&ЭкспортJson` | Включает экспорт в JSON. |
| `&ЭкспортHtml` | Включает экспорт в HTML. |
| `&КаталогАртефактов(<КаталогАртефактов>)` | Задает каталог сохранения экспортируемых файлов. |

```bsl
&МониторингПамяти
&СортировкаОтБыстрыхКМедленным
&КолонкаМин
&КолонкаМакс
&КолонкаПроцентиль(10, 80)
&ЭкспортJson
&КаталогАртефактов("./BenchmarkArtifacts")
Процедура ПриСозданииОбъекта()
КонецПроцедуры
```

Для `&КолонкаПроцентиль` можно указать несколько значений:

```bsl
&КолонкаПроцентиль(10, 50, 90)
Процедура ПриСозданииОбъекта()
КонецПроцедуры
```

Можно использовать и форму без параметров:

```bsl
&КолонкаПроцентиль
Процедура ПриСозданииОбъекта()
КонецПроцедуры
```

Если аннотация присутствует без значений, используется процентиль по умолчанию.

Подробнее: [Мониторинг памяти](МониторингПамяти.md), [Сортировка отчета](СортировкаОтчета.md), [Экспорт результатов](ЭкспортРезультатов.md), [Метрики](Метрики.md).

### `&ИсполняющаяСреда`

**Синтаксис**

`&ИсполняющаяСреда(<Версия>, <Наименование> = "", <ЭтоЭталон> = Ложь)`

Добавляет отдельную runtime-среду для запуска бенчмарков.

**Параметры**

- `Версия` - версия OneScript, алиас (`stable`, `dev`) или абсолютный путь к `oscript`;
- `Наименование` - пользовательское имя среды в отчете;
- `ЭтоЭталон` - признак эталонной среды.

Аннотацию можно указать несколько раз.

```bsl
&ИсполняющаяСреда("stable")
&ИсполняющаяСреда("dev")
Процедура ПриСозданииОбъекта()
КонецПроцедуры
```

```bsl
&ИсполняющаяСреда(
	Версия = "stable",
	Наименование = "Стабильная",
	ЭтоЭталон = Истина
)
Процедура ПриСозданииОбъекта()
КонецПроцедуры
```

Подробнее: [Исполняющие среды](ИсполняющиеСреды.md), [Эталоны](Эталоны.md#эталонные-среды-выполнения).

## Обработчики событий жизненного цикла

Аннотации событий указываются на отдельных экспортных методах.

| Аннотация | Когда вызывается | Что есть в контексте |
| --- | --- | --- |
| `&ПередВсеми` | Один раз перед запуском всех кейсов | `Конфигурация`, `ДескрипторыБенчмарков` |
| `&ПослеВсех` | Один раз после завершения всех кейсов | `Конфигурация`, `ДескрипторыБенчмарков`, `Запуски`, `Отчет`, `СредаОкружения` |
| `&ПередКаждымКейсом` | Перед каждым кейсом | `Кейс` |
| `&ПослеКаждогоКейса` | После каждого кейса | `Кейс`, `Замеры`, `Статистика` |
| `&ПередИтерацией` | Перед каждой итерацией | `Кейс`, `Этап`, `НомерИтерации` |
| `&ПослеИтерации` | После каждой итерации | `Кейс`, `Этап`, `НомерИтерации`, `РезультатИтерации` |

**Синтаксис метода-обработчика**

`Процедура <ИмяМетода>(Контекст) Экспорт`

```bsl
&ПередВсеми
Процедура ПодготовитьДанные(Контекст) Экспорт
КонецПроцедуры

&ПослеВсех
Процедура ОчиститьРесурсы(Контекст) Экспорт
КонецПроцедуры
```

```bsl
&ПередИтерацией
Процедура ПередИтерацией(Контекст) Экспорт
КонецПроцедуры

&ПослеИтерации
Процедура ПослеИтерации(Контекст) Экспорт
КонецПроцедуры
```

Важно:

- при наличии `&ПередИтерацией` или `&ПослеИтерации` количество вызовов за итерацию принудительно становится `1`;
- обработчики жизненного цикла предназначены для подготовки, валидации и постобработки, а не для самой измеряемой логики.

Подробнее: [Обработчики событий](ОбработчикиСобытий.md).
