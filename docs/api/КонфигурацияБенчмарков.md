# КонфигурацияБенчмарков

Класс для программной настройки запуска бенчмарков, параметризации, отчета, экспорта, обработчиков событий и исполняющих сред.

## Синтаксис:

```bsl
Новый КонфигурацияБенчмарков(<Тип>)
```

**Параметры:**

| Имя | Тип | Значение по умолчанию | Описание |
| -- | -- | -- | -- |
| Тип | Тип, Неопределено | `Неопределено` | Тип класса бенчмарков. Если передан, конфигурация заполняется аннотациями этого типа |

## Значения по умолчанию

| Свойство | Значение |
| -- | -- |
| Стратегия | `СтратегииЗапускаБенчмарка.ПропускнаяСпособность` |
| КоличествоИтераций | `15` |
| КоличествоВызововЗаИтерацию | `0` - рассчитывать автоматически |
| КоличествоПрогревочныхИтераций | `6` |
| МинимальноеВремяИтерации | `100` мс |
| МинимальноеКоличествоВызововЗаИтерацию | `4` |
| СортировкаОтчета | `СортировкиОтчетаБенчмарков.ПоУмолчанию` |
| КаталогАртефактов | `.\BenchmarkArtifacts` |
| Мониторинг памяти | `Ложь` |

## Пример

```bsl
Конфигурация = Новый КонфигурацияБенчмарков()
    .УстановитьСтратегию(СтратегииЗапускаБенчмарка.ПропускнаяСпособность)
    .УстановитьКоличествоИтераций(20)
    .УстановитьКоличествоПрогревочныхИтераций(5)
    .ДобавитьПараметр("Размер", 1000)
    .ДобавитьКолонку(КолонкиОтчетаБенчмарков.Мин)
    .ДобавитьЭкспортер(ЭкспортерыРезультатовБенчмарков.Markdown);
```

> [!NOTE]
> Большинство устанавливающих методов возвращают текущую конфигурацию, поэтому их удобно вызывать цепочкой.
